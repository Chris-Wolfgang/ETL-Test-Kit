Examples

public class MyLoaderIdempotencyTests
    : IdempotentLoaderContractTests<MyLoader, MyRecord>
{
    protected override MyLoader CreateSut(int itemCount) =>
        new MyLoader(collectItems: true);

    protected override IReadOnlyList<MyRecord> CreateSourceItems() =>
        new List<MyRecord> { new("a"), new("b"), new("c"), new("d"), new("e") };

    protected override IReadOnlyList<MyRecord>? TryGetLoadedItems(MyLoader sut) =>
        sut.GetCollectedItems();
}

Remarks

This base class is opt-in. Inherit from it only if your loader supports being run more than once on the same instance, processing the full input each time and carrying no state over from a previous run. Single-use components — for example a loader that opens a destination stream once and disposes it after the first load — must not inherit this class.

Each test calls CreateSut(int) exactly once and then loads CreateSourceItems() twice, asserting that the second run is unaffected by the first.

Remarks

The default implementation reads Wolfgang.Etl.Abstractions.LoaderBase<TDestination, TProgress>.CurrentItemCount assuming the loader derives from Wolfgang.Etl.Abstractions.LoaderBase<TDestination, TProgress> with a Wolfgang.Etl.Abstractions.Report progress type, which is the convention for ETL loaders. Override this when the loader exposes its progress count through a different progress type or member.

LoadAsync_when_called_twice_CurrentItemCount_resets_Async()

Verifies that CurrentItemCount reflects only the most recent run: after loading the same source twice into the same instance, the count equals the number of items in a single run, not the cumulative total across both runs.

Remarks

This verifies the per-run reset contract introduced in Wolfgang.Etl.Abstractions 0.14.0 (ETL-Abstractions#246), where CurrentItemCount and CurrentSkippedItemCount reset at the start of each run.

Remarks

Override to return the loader's collected snapshot (for example sut.GetCollectedItems()). When this returns null the no-state-leak test treats the loader as non-collecting and skips its collection check.
