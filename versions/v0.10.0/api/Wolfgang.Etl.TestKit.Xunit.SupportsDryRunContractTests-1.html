Examples

public class MyLoaderDryRunContractTests
    : SupportsDryRunContractTests<MyLoader>
{
    protected override MyLoader CreateSut() => new MyLoader(connectionString);

    protected override async Task<bool> RunAndReportSideEffectAsync(bool isDryRun)
    {
        var loader = new MyLoader(connectionString) { IsDryRun = isDryRun };
        await loader.LoadAsync(CreateSourceItems().ToAsyncEnumerable());
        return await CountRowsAsync(connectionString) > 0;
    }
}

Remarks

Inherit from this class to verify that your stage honours the Wolfgang.Etl.Abstractions.ISupportDryRun contract — that it actually skips its external side effect when Wolfgang.Etl.Abstractions.ISupportDryRun.IsDryRun is true, rather than merely exposing the property. The interface is opt-in and the ETL base classes deliberately do not implement it, so this contract is the only guarantee that an implementer truly performs no write/acknowledgement in dry-run mode.

The base is stage-agnostic: implement RunAndReportSideEffectAsync(bool) to create your stage with the supplied dry-run setting, drive it to completion (a loader's LoadAsync, an extractor's enumerated ExtractAsync, and so on), and report whether the external side effect occurred.

This contract covers Wolfgang.Etl.Abstractions.ISupportDryRun.IsDryRun only. That the pipeline still runs fully in dry-run mode (progress counters advancing, items enumerated) is covered by the stage's own base-class contract tests — for example LoaderBaseContractTests<TSut, TItem, TProgress> — which a dry-run loader should also inherit.
