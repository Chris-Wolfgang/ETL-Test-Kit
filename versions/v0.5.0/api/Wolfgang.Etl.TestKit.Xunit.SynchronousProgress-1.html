Remarks

The standard Progress<T> class captures the current SynchronizationContext and posts callbacks to it asynchronously. In unit tests running without a synchronisation context this means callbacks are dispatched to the thread pool and may not have fired by the time an assertion runs.

SynchronousProgress<T> calls the handler immediately and on the same thread that calls Report(T), making progress assertions in tests fully deterministic without requiring delays or manual synchronisation.
