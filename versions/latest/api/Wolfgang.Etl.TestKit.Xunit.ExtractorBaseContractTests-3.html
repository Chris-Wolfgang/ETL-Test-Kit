Examples

public class MyExtractorContractTests
    : ExtractorBaseContractTests<MyExtractor, MyRecord, MyProgress>
{
    protected override MyExtractor CreateSut(int itemCount) =>
        new MyExtractor("path/to/test-data.csv", itemCount);

    protected override IReadOnlyList<MyRecord> CreateExpectedItems() =>
        new List<MyRecord> { new("a"), new("b"), new("c"), new("d"), new("e") };

    protected override MyExtractor CreateSutWithTimer(IProgressTimer timer) =>
        new MyExtractor("path/to/test-data.csv", timer);
}

Remarks

Inherit from this class to get a comprehensive suite of contract tests covering all public behaviour defined by Wolfgang.Etl.Abstractions.ExtractorBase<TSource, TProgress>:

  • All four ExtractAsync overloads yield the expected items.
  • Cancellation is honoured on all cancellable overloads.
  • CurrentItemCount is incremented as items are yielded.
  • ReportingInterval rejects values less than 1.
  • MaximumItemCount stops extraction at the specified limit.
  • MaximumItemCount rejects values less than 0.
  • SkipItemCount rejects values less than 0.
  • Progress callbacks fire at least once per extraction.
  • CreateProgressReport reflects CurrentItemCount.

You are responsible for implementing MaximumItemCount behaviour in your ExtractWorkerAsync override. The contract tests for Wolfgang.Etl.Abstractions.ExtractorBase<TSource, TProgress>.MaximumItemCount will only pass if your extractor checks that property and stops yielding when the limit is reached.

Returns the expected items that the SUT should yield when created with CreateSut(int). Must return at least 5 items. The first itemCount items from this list are the expected output when CreateSut(itemCount) is called.
